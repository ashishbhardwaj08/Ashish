# Definition

Thickening is the **opposite of thinning**.

> It increases the thickness of objects in a controlled way  
> while preserving their shape and connectivity.

If thinning gives a 1-pixel skeleton,  
thickening builds it outward.

# Basic Idea
Original:

```
  ██
  ██
  ██
```

After thickening:

```
 ████
 ████
 ████
```


# Main Concept

Thickening works using:

- **Hit-or-Miss transformation**
- Adding pixels instead of removing them
- Iterative process

It expands object boundaries in a controlled way.
# Mathematical Formula


$A \odot B = A \cup (A \circledast B)$  


Where:

- (A) = binary image
- (B) = structuring element
- $(\circledast)$ = Hit-or-Miss
- ∪ = union (OR)

Meaning:

1. Find matching pattern (Hit-or-Miss)
2. Add those pixels to original image


# Working

1. Apply Hit-or-Miss
2. Add detected pixels to image
3. Repeat until no change
So instead of deleting boundary pixels (like thinning),  
we **add boundary pixels**.


# Code

Full hit-or-miss thickening is complex,  
but practical thickening can be done like this:

```python
import cv2
import numpy as np

img = cv2.imread("binary.png", 0)
_, binary = cv2.threshold(img, 127, 255, cv2.THRESH_BINARY)

kernel = np.ones((3,3), np.uint8)

thickened = cv2.dilate(binary, kernel, iterations=1)

cv2.imshow("Original", binary)
cv2.imshow("Thickened", thickened)
cv2.waitKey(0)
```

This expands object boundaries.
