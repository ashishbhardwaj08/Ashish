# 1. Feature Vector

###  Definition

A **feature vector** is a numerical representation of an image (or object) as a list of measurable properties.

$\mathbf{f} = [f_1, f_2, f_3, ..., f_n]$ 


Each ( f_i ) = one feature (edge, texture, color, etc.)

---

### Uses

- Image classification
- Object detection
- Face recognition
- Scene understanding

---

### Working

1. Extract features (edges, textures, gradients)
2. Convert into vector form
3. Feed into classifier (like SVM)

---

#### *“Minimal projection is primary projection”*

- We try to **reduce dimensions** while keeping important information
- Example: PCA → projects high-dimensional data to lower dimension
- “Primary projection” = most informative direction

---

# 2. Priming & Biasing

### Definition

- **Priming**: Pre-conditioning system using prior information
- **Biasing**: Giving preference to certain features or outcomes

---

### Example

- Face detection biased toward eye-like patterns
- Neural networks use **bias term** to shift decision boundary

---

### Importance

- Helps reduce computation
- Improves learning speed
- Avoids irrelevant features

---

### Techniques

# 4. Cosine Similarity (VERY IMPORTANT)

### Definition

Measures similarity between two vectors based on **angle**, not magnitude.

$\cos(\theta) = \frac{\mathbf{A} \cdot \mathbf{B}}{||A|| \ ||B||}$

---

#### *“Compare two unit vectors wrt angle”*

- Convert vectors to **unit vectors**
- Then:   
    $\cos(\theta) = A \cdot B$ 

#### Interpretation

- 1 → same direction (very similar)
- 0 → orthogonal (different)
- -1 → opposite

### Use

- Image similarity
- Text similarity
- Feature matching

---

# 6. Gabor Filter (VERY IMPORTANT)

### Definition

A **Gabor filter** is a sinusoidal wave modulated by a Gaussian.

---

### Formula (conceptual)

$G(x,y) = \text{Gaussian} \times \text{Sinusoid}$ 

---

###  Intuition

- Acts like **human visual system**
- Detects:
    
    - Texture
    - Edges
    - Orientation

---

### Parameters

- Scale (frequency)
- Orientation (angle)
- Phase

---

### Your Note Explained
 **“4 scale × 8 direction = 32 features”**

- Apply filters at:
    - 4 different sizes
    - 8 orientations
- Total features = **32 feature maps**

---

### Uses

- Texture classification
- Face recognition
- Scene understanding
---

# 7. Feature Maps

### Definition

Output images after applying filters.

---

### Example

- Input image → Gabor filters → multiple outputs
- Each output = highlights specific pattern
---

### Role

- Captures spatial + frequency information

---

# 8. Bins & Gist Features

### Bins

- Divide feature values into groups
- Used in histograms

---

### Gist Features (IMPORTANT)

### Definition

A **compact representation of a scene** using global features.

---

### Working

1. Apply Gabor filters
2. Divide image into region
3. Compute average responses (bins)
4. Combine into feature vector

---

#### *“Spatial info increases on dividing into parts”*

- Whole image → loses detail
- Divide into grid → captures spatial layout

---

###  Use

- Scene classification
- Fast recognition

---

# 9. SIFT Features (VERY IMPORTANT)

### Definition

Scale-Invariant Feature Transform → detects keypoints invariant to:

- Scale
- Rotation
- Illumination

---

###  Steps

1. Scale-space extrema detection
2. Keypoint localization
3. Orientation assignment
4. Descriptor generation

---

### Output

- Keypoints + descriptors (feature vectors)

---

### Why Powerful?

- Robust matching across images

---

# 11. SVM (Support Vector Machine)

### Definition

Finds **optimal hyperplane** separating classes.

---

### Key Idea

Maximize margin between classes.
### Equation
 
$w \cdot x + b = 0$ 

---

### Concepts

- Support vectors → closest points
- Margin → distance between classes

---

### Advantages

- Works well in high dimensions
- Effective for image classification

---

# 12. Graph Representation of Features (VERY IMPORTANT)

### Definition

Image features are represented as a **graph**:

- Nodes → features
- Edges → relationships

#### *“Feature is represented by a graph”*

- Each feature = node
- Connections = spatial or similarity relations
- 
# 13. Subgraph Matching

#### *Template = features = subgraph*

- Template = small graph
- Image = large graph
- Matching = find subgraph inside graph
### Use

- Object detection
- Pattern recognition

---

# 14. Graph Cut Algorithm (IMPORTANT)

### Definition

Used for **image segmentation** by cutting graph into parts.
### Idea

- Pixels = nodes
- Similar pixels → strong edges
- Cut edges to separate objects
### Objective

Minimize:

- Cut cost
- While preserving similarity
### Uses

- Foreground/background separation
- Image segmentation

---

# 15. Full Pipeline

**“Images → Gist features → Graph representation”**

### Step-by-step:

1. Input Image
2. Extract features (Gabor / SIFT / Gist)
3. Convert to feature vector
4. Represent as graph
5. Use:
    - SVM → classification
    - Graph cut → segmentation
    - Subgraph matching → detection
