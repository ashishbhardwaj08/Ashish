# Feature Extraction, Edges, Corners, and Image Alignment

In computer vision, the goal is not just to look at raw pixels, but to **extract meaningful features** that are stable, distinctive, and useful for tasks like:

- segmentation
- recognition
- panorama stitching
- image registration

---

# 1. Why Feature Extraction?

Raw pixels are:

- noisy
- redundant
- sensitive to lighting and scale

So we extract **features** such as:

- edges
- corners
- blobs

These features summarize important structure in the image.

---

# 2. Edges: Based on Change, Not Value

Important idea:
 **Edges depend on intensity difference, not absolute intensity**

Example:

- Dark-to-light transition → edge
- Even if both regions are bright, a sudden change creates an edge

Mathematically, edges are detected using **derivatives (gradients)**.

---

## Edge Detection as Band-Pass Filtering

- Low frequencies → smooth regions
- High frequencies → noise
- Mid frequencies → edges

Edge detectors behave like **band-pass filters**

They:

- remove very smooth areas
    
- suppress noise
    
- keep sharp transitions
    

---

## Why Linear Filters Cannot Fully Remove Noise

Linear filters (like mean filter):

- blur noise
- but also blur edges


 Trade-off
- remove noise → lose detail
- preserve detail → keep noise

That’s why we use:

- Gaussian smoothing (controlled blur)
- or nonlinear filters (median)

---

# 3. Corners and Blobs (Key Features)

## Corners

A corner is a point where intensity changes in **multiple directions**.

Example:

- intersection of two edges

Why important?

- highly distinctive
- easy to match across images
---

## Blobs

Blobs are regions that differ in intensity from surroundings.

Example:

- bright spot on dark background
Blobs are useful for:

- object detection
- scale detection

---

# 4. Characteristics of Good Features

A good feature should have:

### 1. Repeatability

Same feature should be detected in different images.

### 2. Saliency

Feature should stand out clearly.

### 3. Locality

Should be localized in a small region.

### 4. Compactness

Feature representation should be concise.

### 5. Efficiency

Fast to compute.

---

# 5. Harris Corner Detector (Core Concept)

Corner detection is based on how intensity changes when we shift a window.

---

## Idea: Shift Window by (u, v)

We measure change in intensity:

$E(u,v) = \sum W(x,y) [I(x+u,y+v) - I(x,y)]^2$

If:

- small change → flat region
- change in one direction → edge
- change in all directions → corner

---

## Second Moment Matrix

This leads to matrix:

$$M =  
\begin{bmatrix}  
I_x^2 & I_x I_y \\ 
I_x I_y & I_y^2  
\end{bmatrix} $$ 

This is called the **structure tensor**.

---

## Interpretation

Let eigenvalues be ( \lambda_1, \lambda_2 )

|Case|Meaning|
|---|---|
|both small|flat|
|one large|edge|
|both large|corner|

---

## Harris Response Function
 
$R = \det(M) - k(\text{trace}(M))^2$  

Where:

- (k) ≈ 0.04–0.06

 Large (R) → corner
 
---

## Harris Step-by-Step

1. Compute gradients $(I_x, I_y)$
2. Compute products $(I_x^2, I_y^2, I_x I_y)$
3. Apply Gaussian smoothing
4. Form matrix (M)
5. Compute response (R)
6. Threshold → detect corners

---

## Code (Harris Corner Detection)

```python
import cv2
import numpy as np

img = cv2.imread('image.jpg')
gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)

gray = np.float32(gray)

# Harris corner detection
dst = cv2.cornerHarris(gray, 2, 3, 0.04)

# Mark corners
img[dst > 0.01 * dst.max()] = [0, 0, 255]

cv2.imshow('Corners', img)
cv2.waitKey(0)
cv2.destroyAllWindows()
```

# 6. Invariance in Features

Good features must be invariant to transformations.

---

## Geometric Invariance

- Rotation invariant
- Scale invariant
- Affine invariant

---

## Photometric Invariance

- illumination changes
- brightness/contrast changes

---

# 7. Scale-Space and Scale Normalization

Objects appear at different sizes.

So we analyze image at multiple scales.

---

## Scale Normalization

To make detection consistent:

Multiply derivative by ( \sigma )

This ensures features are comparable across scales.

---

# 8. Blob Detection

Blobs are detected using:

- Laplacian of Gaussian (LoG)
- Difference of Gaussian (DoG)

---

## Scale Selection

We vary ($\sigma$ ) and detect where response is maximum.

That gives:

- blob size
- blob location

---

## Affine Adaptation

Real objects may be stretched or skewed.

Affine adaptation adjusts detection to:

- scale
- orientation
- shape distortion

---

# 9. Image Registration

## Definition

Aligning two images of the same scene.

Example:

- medical scans
- satellite images

---

## Key Idea

To align two images, we need **at least 3 points**.

Why?

Because we need to estimate transformation:

- translation
- rotation
- scaling

3 non-collinear points define transformation uniquely.

---

# 10. Panorama Stitching

Panorama = combining multiple images into one.

Steps:

1. Detect features (corners/blobs)
2. Match features between images
3. Estimate transformation (homography)
4. .Warp images
5. Blend images

---

## Code (Basic Stitching using OpenCV)

```python
import cv2

img1 = cv2.imread('left.jpg')
img2 = cv2.imread('right.jpg')

stitcher = cv2.Stitcher_create()

status, pano = stitcher.stitch([img1, img2])

if status == cv2.Stitcher_OK:
    cv2.imshow('Panorama', pano)
    cv2.waitKey(0)
```

---

# 11. Aspect Ratio

Aspect ratio = width / height

Important because:

- distortion occurs if ratio changes
- features depend on correct geometry

---

# 12. Summary

Pipeline:

1. Remove noise (Gaussian)
2. Detect edges (gradients)
3. Extract features (corners/blobs)
4. Ensure invariance (scale, rotation)
5. Represent in feature space
6. Match features
7. Align images (registration)
8. Stitch images (panorama)

---

# Final Intuition

- Edges → where change happens
- Corners → where change happens in all directions
- Blobs → regions of distinct intensity
- Features → stable points for matching
- Scale-space → handle size variations
- Invariance → handle real-world transformations
- Registration → align images
- Stitching → combine images
