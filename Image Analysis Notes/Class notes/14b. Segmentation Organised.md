# 1. What is Image Segmentation?

## Definition

Image segmentation means **dividing an image into meaningful regions** such that:

- Regions are **homogeneous (similar inside)**
- Regions are **different from each other**
- Regions are **mutually exclusive & totally exhaustive**

## Goal

- Separate **foreground from background**
- Identify different objects/regions
- Simplify image for further processing

## Example

Image → Segmentation:

- face
- clothes
- background

# 3. Threshold-Based Segmentation

## Idea

Divide image based on intensity.


$I(x,y) > T \rightarrow \text{Foreground}$
 
$I(x,y) \leq T \rightarrow \text{Background}$  



## Types

### 1. Global Threshold
Same threshold for entire image

### 2. Adaptive Threshold
Threshold varies locally

### 3. Otsu’s Method
Automatically finds best threshold


## Code

```python
import cv2

# Read image in grayscale
img = cv2.imread('image.jpg', cv2.IMREAD_GRAYSCALE)

# 1. Global Thresholding
_, thresh1 = cv2.threshold(img, 127, 255, cv2.THRESH_BINARY)

# 2. Adaptive Thresholding
thresh2 = cv2.adaptiveThreshold(
    img,
    255,
    cv2.ADAPTIVE_THRESH_MEAN_C,  # or GAUSSIAN_C
    cv2.THRESH_BINARY,
    11,  # block size (must be odd)
    2    # constant subtracted
)

# 3. Otsu’s Thresholding
_, thresh3 = cv2.threshold(img, 0, 255,cv2.THRESH_BINARY + cv2.THRESH_OTSU)


cv2.imshow("Original", img)
cv2.imshow("Global Threshold", thresh1)
cv2.imshow("Adaptive Threshold", thresh2)
cv2.imshow("Otsu Threshold", thresh3)

cv2.waitKey(0)
cv2.destroyAllWindows()
```


## Advantages

- Simple
- Fast

## Disadvantages

- Fails with uneven lighting
- Cannot handle overlapping intensities

# 4. Edge-Based Segmentation

## Core Idea

Segment the image by finding **boundaries (edges)** where intensity changes sharply.
**Edges depend on change (gradient), not absolute values.**


## Mathematical Idea (Gradient)

$|\nabla I| = \sqrt{I_x^2 + I_y^2}$

- $I_x, I_y$​: intensity change in x and y directions
- Large gradient → edge


## Common Edge Operators

- **Sobel** (with smoothing)
- **Prewitt**
- **Laplacian** (2nd derivative)
- **Canny** (best practical method)


##  Why Pre-Smoothing?

Real images have noise → gradients amplify noise.  
So we often do:
Image → **Gaussian blur** → **Edge detection**


##  Edge-Based Segmentation Pipeline

1. Smooth image (Gaussian)
2. Compute gradients
3. Detect edges (thresholding)
4. Link edges (optional)
5. Fill regions (optional)


## Canny Edge Detector (Best)

Steps:

1. Gaussian smoothing
2. Gradient computation
3. Non-maximum suppression
4. Double threshold
5. Edge tracking (hysteresis)



## Code (Canny)
```python
import cv2  
  
img = cv2.imread('image.jpg', 0)  
  
edges = cv2.Canny(img, 100, 200)  
  
cv2.imshow("Edges", edges)  
cv2.waitKey(0)
```



## Advantages

- Good for **boundary detection**
- Simple and fast
- Captures shape information

## Disadvantages

- Edges may be **broken**
- Does NOT directly give regions
- Sensitive to noise

## Key Insight
 Edge-based segmentation finds **where regions change**,  
but not necessarily **what the regions are**.

# 5. Region-Based Segmentation

###  Core Idea

Segment image by **grouping similar neighboring pixels**.

 Pixels in same region must be:

- similar in intensity/color
- spatially connected

### Key Principle

Instead of finding boundaries, we **build regions directly**.

## 5.1 Region Growing (Important)

### Idea

Start from a seed → expand region.

###  Steps

1. Choose seed pixel
2. Check neighbors
3. If similar → add to region
4. Repeat


### Similarity Condition

$|I(x,y) - I(\text{seed})| < T$ 

### Advantages

- Produces **connected regions**
- Uses spatial information

### Disadvantages

- Depends on seed selection
- Sensitive to threshold

---

## 5.2 Split and Merge  (Important)

### Idea

- **Split** if region is not uniform
- **Merge** if neighboring regions are similar

### Steps

1. Start with full image
2. Split non-homogeneous regions
3. Merge similar neighbors
4. Repeat

### Concept

Top-down + bottom-up combination.

### Advantages

- Balanced segmentation
- Works on complex images

### Disadvantages

- Computationally heavy
- Needs good criteria

---

# Region-Based vs Edge-Based

| Feature           | Edge-Based         | Region-Based           |
| ----------------- | ------------------ | ---------------------- |
| Approach          | Detect boundaries  | Grow regions           |
| Uses              | Gradient           | Similarity + neighbors |
| Output            | Edges              | Full regions           |
| Noise sensitivity | High               | Moderate               |
| Completeness      | Incomplete regions | Complete segmentation  |
|                   |                    |                        |

---

# Key Conceptual Difference

- **Edge-based → detects “where change happens”**
- **Region-based → finds “what belongs together”**

# When to Use What?

### Use Edge-Based when:

- You need shape/boundaries
- Objects are well separated

### Use Region-Based when:

- Regions are smooth and connected
- You want full segmentation

- Edge-based = **outline drawing**
- Region-based = **color filling**
## Code

```python
# Region Growing
import cv2
import numpy as np

def region_growing(img, seed, threshold=10):
    h, w = img.shape
    visited = np.zeros((h, w), dtype=bool)
    segmented = np.zeros((h, w), dtype=np.uint8)

    # Stack for DFS
    stack = [seed]

    while stack:
        x, y = stack.pop()

        if visited[x, y]:
            continue

        visited[x, y] = True
        segmented[x, y] = 255

        # Check 8-neighbors
        for dx in [-1, 0, 1]:
            for dy in [-1, 0, 1]:
                nx, ny = x + dx, y + dy

                if 0 <= nx < h and 0 <= ny < w:
                    if not visited[nx, ny]:
                        # Similarity condition
                        if abs(int(img[nx, ny]) - int(img[x, y])) < threshold:
                            stack.append((nx, ny))

    return segmented


# --------- Usage ---------
img = cv2.imread('image.jpg', 0)

seed_point = (100, 100)  # choose manually

result = region_growing(img, seed_point, threshold=8)

cv2.imshow("Original", img)
cv2.imshow("Region Growing", result)
cv2.waitKey(0)
cv2.destroyAllWindows()
```

```python
# Split and Merge
import cv2  
import numpy as np  
  
# Check if region is homogeneous  
def is_homogeneous(region, threshold):  
return np.std(region) < threshold  
  
  
def split_and_merge(img, threshold=10, min_size=8):  
h, w = img.shape  
result = np.zeros_like(img)  
  
def split(x, y, size):  
region = img[x:x+size, y:y+size]  
  
# Stop condition  
if size <= min_size or is_homogeneous(region, threshold):  
result[x:x+size, y:y+size] = np.mean(region)  
return  
  
# Split into 4 parts  
half = size // 2  
split(x, y, half)  
split(x + half, y, half)  
split(x, y + half, half)  
split(x + half, y + half, half)  
  
# Ensure square image  
size = min(h, w)  
size = 2 ** int(np.floor(np.log2(size))) # nearest power of 2  
  
split(0, 0, size)  
  
return result  
  
  
# --------- Usage ---------  
img = cv2.imread('image.jpg', 0)  
  
result = split_and_merge(img, threshold=12, min_size=8)  
  
cv2.imshow("Original", img)  
cv2.imshow("Split & Merge", result.astype(np.uint8))  
cv2.waitKey(0)  
cv2.destroyAllWindows()
```

# 6. Clustering-Based Segmentation

## Idea

Treat segmentation as **clustering in feature space**

## Feature Space

Pixel represented as:

- intensity
- color (R,G,B)
- texture
## 6.1 K-Means Clustering

### Objective

 
$\sum ||x_i - c_j||^2$

## Code

```python
import numpy as np
import cv2

img = cv2.imread('image.jpg')
Z = img.reshape((-1, 3))
Z = np.float32(Z)

K = 3

criteria = (cv2.TERM_CRITERIA_EPS + cv2.TERM_CRITERIA_MAX_ITER, 10, 1.0)

_, labels, centers = cv2.kmeans(Z, K, None, criteria, 10, cv2.KMEANS_RANDOM_CENTERS)

centers = np.uint8(centers)
result = centers[labels.flatten()]
result = result.reshape(img.shape)

cv2.imshow("KMeans", result)
cv2.waitKey(0)
```


## Advantages

- Simple
- Good for color segmentation

## Disadvantages

- Need to choose K
- Sensitive to initialization

#  7. Model-Based Segmentation (Gaussian Mixture Model)(GMM)
## Idea 

Image = mixture of Gaussian distributions

## Intuition

Real images don’t have just one type of pixel distribution.
 Instead, they contain **multiple overlapping groups**:

- sky pixels
- road pixels
- object pixels

Each group can be modeled as a **Gaussian (bell curve)**.

So image = **mixture of Gaussians**


## Mathematical Model

$P(x)=\sum_{k=1}^{K}\pi_k \mathcal{N}(x|\mu_k,\sigma_k^2)$ 
Where:

- K = number of components
- $πk​$= weight (importance)
- μk​ = mean
- σk= variance


## Key Idea

Each pixel has a **probability of belonging to each cluster**.
 Unlike K-means:

- K-means → hard assignment
- GMM → soft assignment


##  Algorithm (EM – Expectation Maximization)

### Step 1: Initialize

- Choose number of Gaussians (K)
- Initialize means and variances

### Step 2: Expectation (E-step)

Compute probability that each pixel belongs to each Gaussian.
### Step 3: Maximization (M-step)
Update:
- means
- variances
- weights
### Step 4: Repeat

Until convrgence
## Output

Each pixel assigned to:

- most probable Gaussian  
    → segmentation
## Advantages

- Handles **overlapping intensities**
- More realistic than K-means
- Probabilistic interpretation

## Disadvantages

- Computationally expensive
- Needs number of components
- Sensitive to initialization

## Why GMM?

- Handles overlapping clusters
- Probabilistic approach

## Code

```python
from sklearn.mixture import GaussianMixture
import cv2
import numpy as np

img = cv2.imread('image.jpg')
Z = img.reshape((-1, 3))

gmm = GaussianMixture(n_components=3)
gmm.fit(Z)

labels = gmm.predict(Z)
segmented = labels.reshape(img.shape[:2])

cv2.imshow("GMM", (segmented * 80).astype('uint8'))
cv2.waitKey(0)
```

# 8. Watershed Segmentation


## Intuition

Think of image as a **landscape**:

- Bright pixels → hills
- Dark pixels → valleys

Now imagine:   Water is poured into valleys

Water fills regions → when two regions meet → **boundary forms**

## Key Idea

- Each region grows like water
- Boundaries form where waters meet

## Steps

### Step 1: Convert image to grayscale

### Step 2: Noise removal

Use:
- Gaussian blur
- Morphological operations
### Step 3: Find markers

Markers define:
- sure foreground
- sure background

### Step 4: Distance transform

Find distance from background → helps separate objects

### Step 5: Apply watershed

Algorithm floods image from markers

## Output

- Regions separated
- Boundaries marked (usually in red)

## Problem: Over-segmentation

Too many small regions may form.
 Solution:

- Use markers properly
- Preprocess image
## Advantages

- Produces **closed boundaries**
- Good for separating touching objects
## Disadvantages

- Sensitive to noise
- Over-segmentation

## Code

```python
import cv2
import numpy as np

img = cv2.imread('image.jpg')
gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)

_, thresh = cv2.threshold(gray, 0, 255, cv2.THRESH_BINARY + cv2.THRESH_OTSU)

kernel = np.ones((3,3), np.uint8)
opening = cv2.morphologyEx(thresh, cv2.MORPH_OPEN, kernel, iterations=2)

dist = cv2.distanceTransform(opening, cv2.DIST_L2, 5)
_, sure_fg = cv2.threshold(dist, 0.7*dist.max(), 255, 0)

markers = np.int32(sure_fg)
markers = cv2.watershed(img, markers)

img[markers == -1] = [0, 0, 255]

cv2.imshow("Watershed", img)
cv2.waitKey(0)
```

# 3. Graph-Based Segmentation

## Intuition

Treat image as a **graph**:
- Pixels → nodes
- Similarity → edges

Goal:  Divide graph into groups (segments)
##  Key Idea

Pixels are connected with weights:

- similar pixels → strong connection
- different pixels → weak connection

Segmentation = **cut weak connections**

## Graph Representation

- Node = pixel
- Edge weight = similarity (e.g., intensity difference)
## Partitioning

We divide graph into groups by minimizing:
 connections between groups  
 maximizing similarity inside group

## Methods

### 1. Minimum Cut

- Cut graph to minimize edge weights
Problem:
- may create very small segments
### 2. Normalized Cut

Improves min-cut by considering:
- size of regions
- balance between groups

## Output

Clusters of pixels = segmented regions
## Advantages

- Uses **global information**
- Strong mathematical foundation
- Works well for complex image
## Disadvantages

- Computationally expensive
- Hard to scale for large images

# 12. Important Concepts

## 12.1 Foreground vs Background

Example:

- Total pixels = 10000
- Foreground = 7000 → 0.7
- Background = 3000 → 0.3

## 12.2 Pixel Neighbourhood

- 4-neighbour
- 8-neighbour
## 12.3 Granularity

- Fine → many small regions
- Coarse → few large regions

## 12.4 Noise

- Most images contain Gaussian noise
- Use smoothing before segmentation

## 12.5 Coherence

Pixels in same region:

- vary smoothly
- not randomly

## 12.6 Feature vs Spatial Info

|Type|Uses|
|---|---|
|Feature|color, intensity|
|Spatial|location, neighbors|

Best segmentation uses both.
