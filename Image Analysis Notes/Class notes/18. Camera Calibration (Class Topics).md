## 1. How images come from the real world

Before any computation, we need a model of how a 3D world becomes a 2D image.

### Context

This appears in **camera modeling, 3D reconstruction, robotics, AR/VR**. Every vision system depends on it.

### Where it is used

- Mapping pixels to real-world coordinates
- Measuring distances from images
- Reconstructing 3D scenes

### Key idea

We assume a projection system:

- **Object plane**: where real-world objects exist (3D)
- **Camera plane**: the optical center / lens system
- **Image plane**: where the image is formed (2D sensor)

### Explanation

Light from a 3D point passes through the camera and hits the image plane.
This gives a mapping:
- 3D point → 2D pixel

This mapping is **not linear in Euclidean space**, which is why we later use homogeneous coordinates.

---

## 2. Relationship between pixels and real-world objects

### Context

This appears in **geometry of vision and measurement problems**.

### Where it is used

- Depth estimation
- Pose estimation
- AR overlays

### Explanation

A pixel is not just a color value—it corresponds to a **ray in 3D space**.

So:

- One pixel → infinite possible 3D points along a ray

To resolve this ambiguity, we need:

- Multiple views
- Or prior knowledge

---

## 3. Correspondence problem

### Context

This is the first major computational challenge in multi-view vision.

### Where it appears

- Stereo vision
- Optical flow
- Structure from motion

### Where it is used

- Depth estimation
- 3D reconstruction
- Motion tracking
### Explanation

We are given two images. We need to find:

- Which pixel in image 1 corresponds to which pixel in image 2

Why this is hard:

- Same object looks different from different angles
- Lighting changes
- Occlusion

Formally:  
We try to match points such that they represent the **same physical point in the world**.

---

## 4. From correspondence to geometry

Once correspondences are found, we can estimate relationships between images.

This leads to:

### Two important matrices

#### (a) Motion matrix

Represents **camera movement** (rotation + translation)

#### (b) Structure matrix

Represents **3D points in the world**

### Context

Appears in **Structure from Motion (SfM)**

### Explanation

We try to factor image observations into:
Image data ≈ Motion × Structure

This is where **matrix factorization** comes in.

---

## 5. Matrix factorization

### Context

Used in **SfM, dimensionality reduction, reconstruction problems**

### Where it is used

- Recovering 3D shape
- Separating motion and structure

### Explanation

We take a measurement matrix (image observations) and decompose it into:

- One matrix for camera motion
- One matrix for scene structure

This works under assumptions like:

- Orthographic projection (simpler case)

---

## 6. Camera calibration

### Context

Before accurate reconstruction, we must know the camera’s internal properties.

### Where it is used

- Converting pixel coordinates to metric units
- Correcting distortions

### Explanation

Calibration finds parameters like:

- Focal length
- Principal point
- Distortion

It defines the exact mapping:  
3D world → 2D image

---

## 7. Why homogeneous coordinates are used

### Context

Projection equations are nonlinear in normal coordinates.

### Where it appears

- Projective geometry
- Camera modeling

### Explanation

To simplify transformations, we use **homogenization**:

We convert:  
(x, y) → (x, y, 1)

Now:

- Projection becomes linear in this space
- Matrix multiplication can represent perspective projection

---

## 8. Constraint-based estimation (why we need multiple points)

### Context

To estimate camera parameters or transformations, we need equations.

### Where it appears

- Calibration
- Homography estimation

### Explanation

Each point correspondence gives equations.

Example idea:

- To estimate a transformation, we need enough independent constraints

Statement:

- “To get 12 parameters, we need 6 points in different planes”

Meaning:

- Each point gives 2 equations
- 6 points → 12 equations → solvable system

Also:

- Points must not be degenerate (e.g., all on one line)

---

## 9. Solving the equations: Least Squares

### Context

Real-world data is noisy, so equations don’t perfectly satisfy constraints.
### Where it is used

- Fitting models
- Estimating transformations

### Explanation

Instead of exact solution, we minimize error:

Minimize:  
difference between predicted and observed values

This is the **least squares method**.

---

## 10. Maximum Likelihood Method

### Context

Adds a probabilistic interpretation to estimation.

### Where it appears

- Statistical vision
- Parameter estimation

### Explanation

We assume:

- Observations contain noise

We choose parameters that maximize:
- Probability of observed data
In many cases:

- Maximum likelihood = least squares (when noise is Gaussian)

---

## 11. Non-linearity in computer vision

### Context

Most real vision problems are not linear.

### Where it appears

- Projection
- Optimization
- Reconstruction

### Explanation

Reasons:

- Perspective projection is nonlinear
- Rotations are nonlinear
- Depth relationships are nonlinear

So:

- We often linearize problems
- Solve iteratively

---

## 12. Curvature, area, and volume analysis

### Context

Once geometry is recovered, we analyze shapes.

### Where it appears

- 3D reconstruction
- Medical imaging
- Surface analysis

### Where it is used

- Object recognition
- Shape comparison

### Explanation

From reconstructed surfaces, we compute:

- Curvature → how surface bends
- Area → surface measurement
- Volume → object size

These help describe **structure beyond just points**.
